# WebApiGame



공부목적 비 실시간 게임 구현



스텔라소라의 참고하여 기능을 조금씩 구현함



## 프로젝트 설정



-asp.net core WebApi



-Nats Jetstream(메시지큐)



-ef core / mssql localdb



-Unity



## 구현기능



-로그인/계정생성



-캐릭터 레벨업



-의뢰 미션



-업적시스템



-메일함



## 프로젝트에서 고민한 부분

-테스트



-메시지큐



-로깅



## 테스트



의미있는 테스트작성을 하기 위해 고민함



기능의 핵심 로직만을 테스트 할 수 있도록 로직과 서버구성을 분리함



## 메시지큐

메시지큐를 게임 서버의 어떤 부분에 적용 시킬 수 있는지 고민함



대개 비동기 처리와 관련된 부분에서 사용가능



이벤트기반 프로그래밍이나 멀티스레딩처럼 사용할 수 있다.



서버장애등을 생각하면 메시지 유실이나 메시지 중복처리에 대해 고려해야함



프로젝트에선 아웃박스 패턴으로 처리



## 로깅

도움되는 로그를 남기는 방법과 로깅 전략에 대해 고민함



나중에 검색할 수 있도록 구조화된데이터를 남기는 것이 중요함



데이터에 일어난 행위를 추적 할 수 있도록 db 업데이트와함께 이벤트(로그)를 남김



데이터 정합성은 db트랜잭션을 통해 보장되고 나중에 이벤트수집기가 배치처리하여 로그를 보냄



로그송신중 에러가 발생해 로그가 중복 송신되어도 EventId를 통해 중복로그를 제거할 수 있다.
